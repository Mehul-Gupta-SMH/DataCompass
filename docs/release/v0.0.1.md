# Poly‑QL v0.0.1 — public preview

Poly‑QL debuts a full-stack natural-language-to-SQL assistant with a two-agent flow, multi-provider LLMs, streaming UX, hardened schema retrieval, and built-in observability.

## What's New
- Two-agent query flow with schema gathering + adaptive re-retrieval, batch schema preload, and clarify-option pills; outputs SQL, Spark SQL, PySpark DataFrame API, or Pandas.
- Provider matrix (OpenAI, Codex, Anthropic, Gemini, GROQ, Claude Code CLI) with balance checks, token tracker, exponential backoff, and live SSE streaming.
- Schema intelligence: ChromaDB embeddings with BM25 + cross-encoder rerank, Kuzu graph for joins, shortest join-path explorer, qualified `database.schema.table` support, and multi-instance routing.
- Ingest Table wizard turns INSERT/CTAS into a reviewable data dictionary, records lineage, and commits to SQLite + ChromaDB + Kuzu in one flow.
- Frontend tabs: chat with outcome badges; interactive ERD; join path + derivative tables; ingest review; provider/model toolbar that greys out invalid keys; auth with username/password or Google SSO.
- Outcome tracker writes every `/api/execute` to JSONL + SQLite, powering validation corpus and UI badges.

## Under the Hood
- FastAPI backend with lifespan startup, JSON structured logging, and Prometheus-compatible metrics (requests, latency histograms, provider call counts, errors).
- Server-Sent Events (`/api/chat/stream`) render incremental LLM tokens in the React UI.
- Retrieval stack: SQLite metadata, ChromaDB vector store, Kuzu graph DB; `_preload_schemas_bulk()` slashes gather round-trips.
- Adaptive re-retrieval agent in `generateQuery` rewrites weak searches up to a configured round cap; stagnation/empty rewrites exit early.
- Robustness: backoff on HTTP 429/5xx, strict JSON parsing with one-shot correction, provider balance probes, and 193-test pytest suite + failure-scenario fixtures; CI runs lint then tests on every PR.

## Known Limitations
- Business semantic layer (glossary-driven term resolution) and rule-based failure classifier are not yet shipped.
- No in-process RAG cache; repeated queries still hit ChromaDB each time.
- Frontend Vitest is not part of CI yet; failures can slip past PR checks.
- `/api/execute` caps results at 500 rows; install DB-specific drivers separately for non-SQLite engines.
- Sessions live in browser `localStorage` (max 30); clearing storage removes history.
